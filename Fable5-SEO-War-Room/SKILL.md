# Fable 5 SEO War Room

Fable 5 is the strategist you can only afford to hire once. This skill gathers your data and compresses it with cheap tools, runs **one** expensive Fable 5 session, and leaves you with 90 days of execution that never needs Fable again.

**The golden rule: Fable never reads raw data.** Raw exports are tens of thousands of rows and would blow the context window and waste Fable credits. Claude gathers everything, compresses it into a tight `BRIEF.md`, and Fable spends its reasoning on strategy, not on reading spreadsheets.

## The flow

```
Gather (manual OR Windsor)  →  Compress (cheap)  →  ONE Fable 5 session  →  Execute 12 weeks (Sonnet/manual)
   GSC + GA4 + DataForSEO         BRIEF.md            BATTLE_PLAN.md            WEEKLY_CHECKLIST.md
```

## Folder layout

```
fable5-war-room/
├── SKILL.md                 # this file
├── config.json              # your domain, competitors, goal, weekly capacity
├── data/                    # raw data lands here (either path)
│   ├── gsc/                 # Search Console
│   ├── ga4/                 # GA4
│   └── dataforseo/          # competitor keywords / gaps / backlinks
├── templates/
│   ├── master_prompt.md         # the one Fable 5 prompt
│   └── dashboard_template.html  # data-driven battle-plan dashboard (do not redesign)
└── output/
    ├── BRIEF.md             # generated: the compressed brief Fable reads
    ├── BATTLE_PLAN.md       # generated by Fable
    ├── WEEKLY_CHECKLIST.md  # generated by Fable
    └── BATTLE_PLAN.html     # generated: interactive dashboard of the whole plan
```

## Setup (once)

Edit `config.json`:
```json
{
  "domain": "yoursite.com",
  "competitors": ["competitor1.com", "competitor2.com", "competitor3.com"],
  "goal": "Book more discovery calls from organic search",
  "weekly_capacity_hours": 5
}
```

## Step 1 — Get your data in (pick ONE path)

You need three sources: **Search Console** (what you rank for), **GA4** (what converts), and **DataForSEO** (what competitors rank for that you don't). There are two ways to get them in.

### Path A — Manual drop (no logins needed by Claude)
Export each platform yourself and drop the raw files into `data/`:
- **Search Console** → Performance → last 16 months → add Queries and Pages dimensions → Export CSV → `data/gsc/`
- **GA4** → Reports → Landing page + Traffic acquisition (include your conversion event) → Export CSV → `data/ga4/`
- **DataForSEO** → export your competitors' ranked keywords, the keyword gap, and a backlink summary → `data/dataforseo/`

Then tell Claude: *"The raw data is in the data/ folder, compress it."*

### Path B — Let Claude pull it
- **Search Console + GA4:** connect them once through **Windsor.ai** and tell Claude to pull GSC and GA4 for your domain. Windsor returns them normalized into `data/gsc/` and `data/ga4/`.
- **DataForSEO:** Claude already knows how to query DataForSEO directly (via the DataForSEO MCP). Just tell it: *"Using the competitors in config.json, pull their ranked keywords, the keyword gap vs my domain, and a backlink summary into data/dataforseo/."*

> Note: Windsor.ai has connectors for Search Console and GA4 but **not** DataForSEO, which is why DataForSEO is pulled straight through Claude's own DataForSEO access. You can also mix paths (e.g. Windsor for GSC/GA4, Claude for DataForSEO).

## Step 2 — Compress (the step that makes one session possible)

Tell Claude (using a normal/cheap model, NOT Fable):

> "Read everything in `data/` and write `output/BRIEF.md`. Distill it down to: my striking-distance keywords (positions 4-20), my decaying pages, my top converting pages from GA4, and a competitor gap table (keywords each competitor ranks for that I don't, with their backlink strength). Pin my goal and weekly capacity from config.json at the top. Keep it tight enough to read in a couple of minutes."

**Sanity check:** open `output/BRIEF.md`. If it's still huge, tell Claude to tighten the thresholds before you spend Fable credits.

## Step 3 — The ONE Fable 5 session

1. Switch your model to **Fable 5**.
2. Open `templates/master_prompt.md` and paste the prompt.
3. Fable reads `output/BRIEF.md`, tells you the 3-5 things it notices, and **asks clarifying questions** — answer them.
4. **Push back on at least one priority** ("why is bet 1 above bet 2?") and let it defend or revise.
5. It writes `output/BATTLE_PLAN.md` and `output/WEEKLY_CHECKLIST.md`.

That's the only time you spend Fable credits.

## Step 4 — Build the dashboard (back on a normal model)

Turn the plan into an interactive dashboard. Tell Claude (normal/cheap model, NOT Fable):

> "Open `templates/dashboard_template.html`. Populate ONLY the `DATA` object at the top of the script using `output/BRIEF.md`, `output/BATTLE_PLAN.md` and `output/WEEKLY_CHECKLIST.md` — every stat, finding, bet, chart row, SERP snapshot, query row, and weekly task. Do not redesign the page or touch anything below the DATA object. Save the result as `output/BATTLE_PLAN.html` and open it."

You get a 5-tab interactive report: Overview (animated stats + traffic-leak funnel + ranking distribution), Findings (expandable evidence), Battle Plan (expandable strategies with KPIs), The Data (hover charts + searchable query table), and a clickable 12-week checklist that saves your progress in the browser.

## Step 5 — Execute for 90 days (no Fable)

Work `WEEKLY_CHECKLIST.md` top to bottom. Each task is tagged:
- **`[Sonnet]`** — hand it to a cheaper model (draft the post, write the schema, generate meta descriptions).
- **`[Manual]`** — you do it (publish, outreach, technical fixes).

You don't open Fable again until the next quarter, when you re-run the flow to re-plan.

## Optional — the comparison beat (great for demos)
Before the Fable session, run the same `BRIEF.md` through a normal model and note how generic the plan is ("write more content, build backlinks"). Then run Fable and watch it catch cross-source insights the generic plan missed. That contrast is the proof the expensive model earns its keep.
